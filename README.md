# Julia Bunyakova — сайт и система управления галереей

Профессиональный трёхъязычный сайт художницы с полноценной CMS: публичная галерея,
блог, комментарии с модерацией, лайки, встроенная аналитика с графиками и картой мира,
медиатека и тёмная админ-панель коммерческого уровня.

**Стек:** Next.js 14 (App Router) · TypeScript · Tailwind CSS · Prisma ORM ·
SQLite (локально) / PostgreSQL (продакшен) · Chart.js · JWT-сессии (jose) · bcryptjs

## Быстрый старт

Требования: **Node.js 18.17+** (рекомендуется 20 LTS) и npm.

```bash
npm install     # установка зависимостей (+ автоматическая генерация Prisma Client)
npm run dev     # запуск в режиме разработки → http://localhost:3000
```

Для продакшен-запуска на собственном сервере:

```bash
npm install
npm run build
npm start       # → http://localhost:3000
```

**База данных создаётся и наполняется автоматически.** Перед каждым `dev` / `build` /
`start` выполняется `scripts/init-db.mjs`, который:

1. создаёт `.env` из `.env.example` (с уникальным `SESSION_SECRET`), если его нет;
2. синхронизирует схему Prisma с базой (`prisma db push`);
3. наполняет базу стартовым контентом (галерея, блог, биография, SEO, переводы).

Сидинг **идемпотентен**: повторные запуски ничего не перезаписывают и не трогают
правки, сделанные через админку.

## Запуск на Open Server Panel

1. Установите Node.js 20 LTS (модуль Node.js в OSP или системная установка).
2. Скопируйте папку проекта, например в `C:\OSPanel\domains\julia-cms`.
3. В консоли OSP выполните:
   ```bash
   cd C:\OSPanel\domains\julia-cms
   npm install
   npm run dev        # разработка, или:
   npm run build && npm start   # продакшен
   ```
4. Сайт доступен на `http://localhost:3000`. Никакой настройки БД не требуется —
   используется SQLite-файл `prisma/dev.db`, который создаётся автоматически.
5. Чтобы открыть сайт на своём домене OSP, настройте reverse-proxy домена
   на `localhost:3000` (Domains → проксирование) либо используйте порт напрямую.

## Деплой на Vercel

На Vercel нужна внешняя PostgreSQL-база: **Vercel Postgres**, [Neon](https://neon.tech)
или [Supabase](https://supabase.com) (у всех есть бесплатный план).

1. Загрузите проект в GitHub и импортируйте репозиторий в Vercel.
2. `vercel.json` уже настроен: команда сборки `npm run build:vercel` автоматически
   подменяет SQLite-схему на PostgreSQL (`prisma/schema.postgres.prisma`), накатывает
   схему на базу и выполняет сидинг.
3. В **Settings → Environment Variables** добавьте:

   | Переменная | Значение |
   |---|---|
   | `DATABASE_URL` | `postgresql://user:password@host:5432/db?sslmode=require` |
   | `SESSION_SECRET` | случайная строка: `openssl rand -hex 32` |
   | `NEXT_PUBLIC_SITE_URL` | `https://ваш-домен.vercel.app` |

4. Deploy. Админка — `https://ваш-домен/admin`.

> **Медиатека на Vercel.** Файловая система Vercel неперезаписываемая, поэтому
> загрузка файлов в `public/uploads` там отключена (API вернёт 501 с пояснением).
> Используйте прямые URL изображений (поле «URL изображения» есть во всех формах)
> или подключите Vercel Blob / S3. Локально и на Open Server загрузка файлов
> работает полностью.

## Переменные окружения

См. `.env.example`:

| Переменная | Описание |

## Мультиязычность

- Языки: **русский (по умолчанию), английский, китайский**. Переключатель в шапке.
- Выбор хранится в cookie `jb_locale`, работает на всех страницах, в SEO и системных сообщениях.
- Базовые словари: `src/locales/{ru,en,zh}.json`. Правки через админку («Переводы»)
  сохраняются в БД как переопределения и имеют приоритет над файлами.
- Контент (картины, статьи, биография) хранится с отдельными полями для каждого языка.

## Аналитика

Встроенная, без внешних сервисов и без персональных данных:

- лёгкий клиентский трекер (`AnalyticsTracker`) фиксирует просмотры страниц
  и длительность визита (через `sendBeacon`);
- посетители считаются по анонимному cookie `jb_visitor` (случайный ID);
- IP не хранится — только усечённый хэш для защиты от накруток;
- страна определяется по заголовкам CDN (на Vercel — автоматически);
- админка и боты в статистику не попадают.

## Безопасность

- **Пароли** — bcrypt, 12 раундов.
- **Сессии** — подписанный JWT в httpOnly-cookie (`SameSite=Strict`, 12 часов).
- **Защита маршрутов** — `middleware.ts` закрывает `/admin/*` и `/api/admin/*`.
- **CSRF** — проверка Origin/Referer для всех изменяющих запросов.
- **Rate limiting** — вход (5/мин), комментарии (5/10 мин), лайки (20/мин) по IP-хэшу.
- **XSS** — санитизация и ограничение длины всех входных строк; React-экранирование вывода.
- **Анти-накрутка лайков** — visitor-cookie + уникальный индекс в БД + rate limit.
- **Комментарии** — публикуются только после ручной модерации.
- **Принудительная смена** первичного пароля при первом входе.

```
julia-cms/
├── prisma/
│   ├── schema.prisma            # схема БД (SQLite — локально)

└── .env.example
```

## API

Публичные: `POST /api/track`, `POST /api/track/duration`, `GET|POST /api/likes`,
`POST /api/comments`.
Авторизация: `POST /api/auth/login`, `POST /api/auth/logout`, `POST /api/auth/change-password`.
Админские (только с сессией): `/api/admin/artworks`, `/categories`, `/posts`, `/comments`,
`/social`, `/biography`, `/settings`, `/translations`, `/seo`, `/media`, `/analytics`.

## Частые вопросы

**Забыл пароль администратора.** Удалите файл `prisma/dev.db` и перезапустите
(`npm run dev`) — база пересоздастся с первичным паролем. На PostgreSQL: удалите
строку из таблицы `User` и выполните `npm run db:seed`.

**Как сбросить базу к демо-контенту?** Локально — удалить `prisma/dev.db` и перезапустить.

**Как открыть базу глазами?** `npm run db:studio` (Prisma Studio).

**Порт 3000 занят.** `PORT=3001 npm start` (Linux/macOS) или `set PORT=3001 && npm start` (Windows).
